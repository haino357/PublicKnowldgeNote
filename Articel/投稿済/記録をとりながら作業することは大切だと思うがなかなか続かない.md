# 記録をとりながら作業することは大切だと思うがなかなか続かない


## はじめに
作業をしている最中は、自分が何を考えて、どこで迷って、どう判断したのかをわかっているつもりでいる。

ただ、あとから見返すと「あの時、何をしていたんだっけ？」となることが多い。作業の結果だけは残っていても、その途中にあった試行錯誤や判断の理由は、意外なほど簡単に抜け落ちてしまう。

だから、作業は記録を取りながら進めた方がいい。そう思ってはいる。思ってはいるのだけれど、これがなかなか続かない。

## 大切だとわかっていても続かない
記録が続かない理由はいくつかある。

- 記録が「本作業ではない余計な作業」に感じられる
- 作業中は頭の中でわかっているので、書く必要を感じにくい
- 没頭している時ほど、記録のために立ち止まるのが面倒になる
- 何を書けばいいか曖昧だと、記録そのものが判断作業になる
- きれいに書こうとして、メモではなく文章作成になってしまう
- 過去に記録を見返さなかった経験があると、「どうせ使わない」と感じる
- 迷い、失敗、途中経過を残すことに少し抵抗がある

記録する利点は、長期的にはかなり大きい。あとで再開しやすくなるし、自分がどこで詰まったのかも見える。判断の履歴が残っていれば、同じところで何度も悩まずに済む。

それでも、その場では「めんどくさい」が勝つ。記録の利益は未来にあるのに、記録の面倒さは今すぐ発生するからだ。

## 記録を作業に割り込ませない
記録を続けるには、立派な作業ログを作ろうとしない方がいい。

作業中に必要なのは、きれいな文章ではなく、あとで戻ってこられる最低限の印だ。作業の流れを止めてまで丁寧に書こうとすると、それ自体が負担になる。

だから、まずは書く場所を1つに決める。日次ノートでも、作業ログでも、チケットのコメントでもいい。迷わず追記できる場所があれば十分だ。

次に、書く項目を絞る。たとえば、次の3つだけでいい。

```md
- 今:
- 分かった:
- 次:
```

もっと削るなら、これだけでもいい。

```md
次:
```

記録の目的は、作業中の自分を実況することではない。未来の自分が再開しやすくなるように、最低限の手がかりを残すことだ。

## 集中している時ほど短く書く
集中している時や、時間が惜しい時に、細かい記録を続けるのは難しい。

そういう時は、記録の粒度を思い切って下げる。たとえば、次のような1行だけでいい。

```md
- 今: APIのエラー原因を確認中
- 次: レスポンスの中身を見る
- 疑問: 認証まわりで失敗している？
```

これくらいなら、作業を完全に止めなくても残せる。完璧な文章ではないが、あとで見返した時に現在地はわかる。

記録するタイミングも、時間で区切るより、作業の節目に合わせた方がいい。ファイルを切り替える時、検索を始める時、方針を変える時、詰まって手が止まった時。そういう自然に流れが切れる瞬間に、1行だけ残す。

作業中は雑でいい。整えるのはあとでいい。集中している時の記録は、文章ではなく栞に近い。

## 作業サイクルに組み込む
記録を習慣にするなら、作業とは別の行為として扱わない方がいい。作業サイクルの中に最初から組み込んでしまう。

たとえば、次のような形にする。

```md
1. 20分作業する
2. 5分だけ記録する
3. 5分休憩する
```

20分くらいであれば、何を考えていたかをまだ思い出せる。5分あれば、きれいな文章までは無理でも、次に再開するための情報は残せる。

大事なのは、作業が終わってから全部思い出そうとしないことだ。長時間作業したあとにまとめて記録しようとすると、細かい判断や迷いはほとんど消えている。だから、短い単位で区切って、その場で少しだけ残す。

## 結局は習慣にすることかな
どんなに方法を考えても、習慣にならなければ絵に描いた餅になってしまう。

ただ、最初から完璧なログを目指す必要はない。むしろ、完璧に書こうとするほど続かない。まずは、作業開始時に目的を書く。詰まったら状況を書く。終わる前に次の一手を書く。この3つだけで十分だと思う。

記録は成果物ではない。作業を再開しやすくするための足場だ。そう考えると、少しだけ負担が軽くなる。

## AIで自動化できるのでは？
最近は成果物を作るのにAIを使うことも増えているので、作業記録もある程度は自動化できるのではないかと思っている。

たとえば、作業内容をまとめるSkillのようなものを作って、一定のタイミングで実行する。開いているファイル、変更した内容、会話のログ、実行したコマンドなどをもとに、その日の作業メモを半自動で作る。

それだけでも、記録の負担はかなり下がるはずだ。最終的には、自分で毎回書くのではなく、作業の流れから自然にログが残る環境を作りたい。

## ゆくゆくは？
作業記録がきちんと残っていれば、自分の考え方や判断の癖も少しずつ見えてくる。

そこまで残せるようになると、単なる備忘録ではなく、自分の作業スタイルを再利用するための材料になる。ゆくゆくは、作業記録から自分の分身のようなものを作って、生産性を何倍にもできる世界を目指したい。

そのためにも、まずは大げさな記録ではなく、作業中に残せる小さな記録から始めるのがよさそうだ。
